Claude Code Desktop

The Code Tab in the Claude Desktop App

A graphical interface for Claude Code — visual diff review, parallel Git-worktree sessions, app previews, PR monitoring, and computer use. Same engine as the CLI.

Code Tab macOS & Windows Visual Diffs

Companion to 11 (Desktop), 13 (Cowork), 14 (Chat)

01

What Is It?

“Claude Code Desktop” is the Code tab inside the Claude Desktop app. The app has three tabs:

Chat — deck 14

  • Conversation, Projects, Artifacts
  • No file-write access
  • Same backend as claude.ai

Code « this deck

  • Interactive coding assistant
  • Direct access to local files
  • You review and approve each change
  • Same engine as the CLI

Cowork — deck 13

  • Autonomous background agent
  • Runs in a cloud VM
  • Hosts Dispatch (task from your phone)

Desktop adds: visual diff review, live app preview, computer use, PR monitoring with auto-fix/auto-merge, parallel sessions with Git worktrees, Dispatch, scheduled tasks, connectors, and Local/SSH/Cloud environments.

Download

All installers come from claude.ai/api/desktop/… via links on code.claude.com/docs/en/desktop-quickstart. No Homebrew, winget, or Linux packages are documented.

Subscription

Requires a Pro, Max, Team, or Enterprise plan. Clicking the Code tab on a free plan prompts you to upgrade.

Cowork Tab Caveat

The Cowork tab requires Apple Silicon (M1 or later) on macOS. Intel Macs still get Chat and Code.

03

First Launch

  1. Run the installer
  2. Launch from Applications / Start menu
  3. Sign in with your Anthropic account
  4. Click the Code tab at the top center
  5. If you get a 403, sign out and back in from the app menu

CLI Is Separate

The desktop app includes Claude Code internally — you don’t need Node.js or the CLI for the Code tab. To use claude from the terminal as well, install the CLI separately.

Windows: Git Required

Git is required for the Code tab to start local sessions on Windows. Install Git for Windows and restart the app.

04

Starting a Session

Configure four things before sending your first message:

1. Environment

  • Local — your machine, direct file access
  • Remote — Anthropic cloud, continues if app closed
  • SSH — remote machine you manage

2. Project Folder

Click Select folder. Remote sessions support multiple repositories via the + button next to the repo pill.

3. Model

Dropdown next to the send button. Opus, Sonnet, Haiku. Locked once the session starts.

4. Permission Mode

Mode selector next to the send button. Can be changed during the session. Defaults to Ask permissions.

05

Permission Modes

Auto accept editsacceptEdits

Ask & Auto Accept

  • Ask: shows a diff, approve or reject each change
  • Auto accept: auto-accepts file edits and common fs commands (mkdir, touch, mv)

Plan Mode

Reads files and explores but does not edit source code. Proposes a plan first. Good before a large refactor.

Auto Mode

Research preview. Team, Enterprise, and API plans only. Requires Sonnet 4.6 or Opus 4.6. Enable in Settings → Claude Code.

Bypass Permissions

No prompts. Equivalent to --dangerously-skip-permissions. Enable in Settings → Claude Code → “Allow bypass permissions mode”. Enterprise admins can disable this.

06

Diff View & Comments

  • After Claude edits files, a +12 -1 indicator appears
  • Click it to open the diff viewer — file list on the left, changes on the right
  • Click any line to open a comment box
  • Submit all comments: Cmd+Enter (macOS) / Ctrl+Enter (Windows)
  • Claude reads the comments and revises

Review Code Button

Top-right of the diff view. Claude evaluates current diffs and leaves inline suggestions. Focuses on compile errors, logic errors, security issues, and obvious bugs — not style, formatting, or lint-level issues.

Add Context

  • @filename with autocomplete (local/SSH only)
  • Attach images, PDFs via button
  • Drag and drop into the prompt
07

Parallel Sessions

Click + New session in the sidebar to work on multiple tasks in parallel.

Automatic Git Worktrees

  • For Git repos, each session gets its own isolated copy
  • Default location: <project-root>/.claude/worktrees/
  • Custom location: Settings → Claude Code → “Worktree location”
  • Branch prefix option
  • Archive icon in sidebar to remove a worktree

Context Compaction

When context fills up, Claude automatically summarizes and continues. Type /compact to trigger earlier.

Continue in Another Surface

From the VS Code icon in the bottom-right: move to Claude Code on the Web (clean working tree required; not for SSH) or open in Your IDE at the current working directory.

08

Environments

Local

  • Runs on your machine, direct file access
  • PATH and a fixed set of Claude Code vars read from shell profile on macOS
  • Set other env vars via the local environment editor (gear icon on Local)

Remote (Cloud)

  • Runs on Anthropic’s cloud
  • Continues even if you close the app
  • Counts toward subscription plan limits — no separate compute charges
  • Monitor from claude.ai/code or the Claude iOS app
  • Supports multiple repositories per session

SSH

  • Environment dropdown → + Add SSH connection
  • Fields: Name, SSH Host, SSH Port (default 22), Identity File
  • Claude Code must be installed on the remote machine
  • Supports permission modes, connectors, plugins, MCP
09

App Preview & Auto-Verify

Claude can start a dev server and open an embedded browser to verify its changes. Works for frontend and backend servers.

Preview Panel

  • Interact with your running app
  • Watch Claude screenshot, click, fill forms, fix issues
  • Start/stop servers from the Preview dropdown
  • Persist cookies and local storage across restarts

Auto-Verify

On by default. Claude verifies changes after every edit — screenshots, error checks, confirmation. Disable per-project with "autoVerify": false in .claude/launch.json.

Configuration File

.claude/launch.json with JSON-with-comments. Fields include runtimeExecutable, runtimeArgs, port, cwd, env, autoPort, and program/args.

10

Pull Request Monitoring

After opening a pull request, a CI status bar appears in the session. Claude Code uses the gh CLI to poll check results.

Auto-Fix

Toggle in the CI status bar. Claude reads failure output and iterates to fix failing CI checks.

Auto-Merge

Toggle in the CI status bar. Merges the PR once all checks pass. Merge method is squash. Must also be enabled in your GitHub repository settings.

Prerequisite

Requires the GitHub CLI (gh) installed and authenticated. If not installed, Desktop prompts you to install it the first time you try to create a PR.

Notification

Claude Code sends a desktop notification when CI finishes.

11

Computer Use

Lets Claude open apps, control your screen, and work the way you would — native apps, mobile simulators, GUIs without an API.

Availability

  • Research preview on macOS and Windows
  • Pro or Max plan only — not on Team/Enterprise
  • Off by default — enable in Settings → General → Computer use
  • macOS also needs Accessibility + Screen Recording

Tool Choice Order

  1. Connector if one exists
  2. Bash for shell commands
  3. Claude in Chrome for browser work
  4. Computer use as the fallback

Per-App Tiers

  • View only — browsers, trading platforms
  • Click only — terminals, IDEs
  • Full control — everything else

Connectors, Plugins, Skills, MCP

Connectors

  • + next to prompt box → Connectors
  • Google Calendar, Slack, GitHub, Linear, Notion, more
  • Graphical setup — MCP servers under the hood
  • Not available in remote sessions

Plugins

  • + → Plugins — browse, install, manage
  • Marketplaces including the official Anthropic marketplace
  • User, project, or local-only scope
  • Not available for remote sessions

Skills / Slash Commands

  • Type / or click + → Slash commands
  • Built-in commands, custom skills, project skills, plugin skills

MCP Servers

  • Supports stdio (local) and Streamable HTTP (HTTP or HTTPS)
  • Configure in ~/.claude.json or .mcp.json
  • Connects from your machine — plain HTTP & private IPs work
  • Chat-tab config (claude_desktop_config.json) is separate
13

Coming from the CLI

Desktop and CLI run the same engine. On macOS, they share config files. On Windows, the MSIX installer sandboxes Desktop — config is not shared.

macOS — Shared

  • CLAUDE.md, CLAUDE.local.md
  • MCP servers in ~/.claude.json, .mcp.json
  • Hooks, skills, settings

Windows — Not Shared

  • MSIX sandboxes Desktop’s filesystem
  • CLI writes to %USERPROFILE%\.claude.json (real fs)
  • Desktop reads from a virtualized path inside %LOCALAPPDATA%\Packages\Claude_…\
  • claude mcp add in PowerShell does not reach Desktop
  • Add MCP servers via Desktop’s own UI instead
  • WSL2 adds a third separate config surface

Move CLI → Desktop

Run /desktop in the terminal. Claude saves the session and opens it in the desktop app, then exits the CLI. macOS and Windows only.

When to Use Which (from the docs)

“Use Desktop when you want visual diff review, file attachments, or session management in a sidebar. Use the CLI when you need scripting, automation, third-party providers, or prefer a terminal workflow.”

14

Not in Desktop & Troubleshooting

Not Available in Desktop

  • Linux — macOS and Windows only
  • Third-party providers — Bedrock, Vertex, Foundry
  • Inline suggestions
  • Agent teams (multi-agent)
  • Scripting / --print / Agent SDK headless
  • --allowedTools / --disallowedTools
  • dontAsk mode (CLI only)

Common Fixes

  • 403 / auth → sign out and back in from the app menu
  • Blank screen → restart (auto-updates on launch)
  • “Git is required” on Windows → install Git for Windows, restart
  • Can’t find npm/node → check shell profile PATH, restart app
