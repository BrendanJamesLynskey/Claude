Claude Code Desktop

The Native GUI for Agentic Coding

The same agent as the CLI, wrapped in a native desktop app — sessions, diffs, permissions, and MCP servers in a single window.

Desktop App Agentic macOS · Win · Linux
01

What Is Claude Code Desktop?

A native application that wraps the Claude Code agent in a graphical interface. Not a different agent — the same Claude Code you know from the CLI, with a GUI layered on top.

Same Core, New Surface

  • Identical agent to the CLI
  • Same tools (Read, Edit, Bash, Grep, Glob)
  • Same hooks, skills, sub-agents
  • Same MCP server ecosystem

What’s New

  • Native windowing & multi-session layout
  • Visual diff viewer for every edit
  • Permission prompts as OS-native dialogs
  • Workspace switcher

What It Is Not

  • Not an IDE — doesn’t edit files by hand
  • Not Claude.ai — runs locally
  • Not a replacement for the CLI — both coexist
02

Why a Desktop App?

Limits of the Terminal UX

  • Diff review painful with wrapped lines
  • Managing parallel sessions needs tmux
  • Permission prompts break typing flow
  • Copy/paste across sessions clumsy
  • Not every developer is a terminal power user

What the GUI Unlocks

  • Side-by-side diff view with syntax highlighting
  • Multiple sessions in tabs or panes
  • OS-native notifications on long tasks
  • Searchable session history

Design Principle

Never hides the underlying agent. Every action is still a tool call; every edit is reviewable; every permission is explicit.

First Launch & Auth

Onboarding Flow

  1. Welcome & product tour
  2. Auth: Claude.ai OAuth, API key, or SSO
  3. Permission grants (filesystem, notifications)
  4. Pick initial workspace
  5. Drop into an empty session

Credential Storage

  • macOS → Keychain
  • Windows → Credential Manager
  • Linux → libsecret
  • Never in plaintext config

Billing

  • Included with Claude Pro / Max / Team / Enterprise
  • Or attach an API key billed per-token
  • Mixed mode supported per workspace

The Main Interface

Three-pane layout.

Sidebar (left)

  • Workspace picker
  • Recent sessions
  • MCP servers, skills, sub-agents, hooks
  • Settings & logs

Transcript (center)

  • Messages, tool calls, permission prompts
  • Plan mode toggle in header
  • Input box at the bottom

Inspector (right)

  • Live diff viewer
  • Accept / Revert buttons
  • Session status (model, context, cost, duration)
  • Transcript export
06

Workspaces & Projects

A workspace is a named directory Claude treats as a project root. Each has its own CLAUDE.md, settings.json, MCP config, and session history.

Worktree-Aware

Git worktrees surface as sibling workspaces — same repo, different branches, parallel agents.

Session History

Every session saved locally with full transcript, tool calls, diffs. Searchable across workspaces, exportable as JSON or Markdown.

Resume & Multi-Session

Reopen a session and the full context rebuilds. Split view runs two sessions side-by-side in the same workspace.

07

Diff Viewer & Review

Every edit appears in the Inspector as a side-by-side diff with syntax highlighting. One of the biggest wins over the CLI.

Review Actions

  • Accept — commit edit to disk
  • Revert — undo
  • Refine — ask Claude to modify the proposal
  • Open in Editor — hand off to VS Code, Cursor

Batch Review

Multi-file changes review one at a time. Accept-all, accept-subset, revert-all. Syntax highlighting via Monaco (same as VS Code).

Git Integration

Stage hunks, generate commit messages, open PRs without leaving the app.

08

Plan Mode & Permissions

Plan Mode

The PLAN toggle enters a read-only mode — Claude can investigate but cannot modify anything. Useful for surveying before acting.

Permission Levels

  • Allow once — one-shot approval
  • Allow always — persist this session
  • Deny — block and explain why

Danger Warnings

Red banners for destructive ops: rm -rf, force-push, DROP TABLE. Explicit confirm required even in Trusted mode.

09

MCP Servers

Full MCP host. What’s new is the GUI around server management.

MCP Server Panel

  • List configured servers with health indicators
  • Enable / disable per workspace
  • Browse tools and resources
  • Live logs for debugging
  • Install from MCP Registry

OAuth Flows

Remote MCP servers open browser for OAuth 2.1 consent. Tokens stored in OS keychain, refreshed automatically.

Config Portability

MCP config lives in .mcp.json — same as the CLI. Works identically whichever surface you use.

10

Sub-agents, Skills, Hooks

Sub-agents

  • Same definitions as CLI
  • Visual graph of parent/child calls
  • Drill into any sub-agent’s transcript

Skills

  • Skills library UI with tags
  • Per-workspace or global enable/disable
  • Marketplace install & author mode

Hooks

  • Visual hook manager in settings
  • Test hook firing from the UI
  • Same shell-command contract as CLI

All live in the same files as the CLI. The desktop app is a view onto your config, not a parallel system.

12

Desktop ↔ CLI ↔ Cloud

Three surfaces, one agent.

Desktop

Native GUI — sessions, diffs, MCP UI, visual review.

CLI

Terminal — scripting, CI/CD, SSH, headless automation.

Cloud

Managed agents — PR reviews, issue triage, long-running tasks via GitHub App.

Shared Config

~/.claude/settings.json, CLAUDE.md, .mcp.json, agents, credentials — all three surfaces read the same files. Start in desktop, resume in CLI, hand off to cloud.

14

Limits & Closing

Known Limitations

  • No headless mode — use the CLI for automation
  • Workspace must be local (SSH via CLI)
  • Heavier startup than the CLI
  • Updates lag the CLI by a few days

Avoid It If…

  • You live in tmux
  • Primarily remote dev boxes
  • Unattended automation

It Shines When…

  • You want visual diff review
  • Managing parallel projects
  • Collaborating with non-terminal users
  • Browsing MCP servers visually
