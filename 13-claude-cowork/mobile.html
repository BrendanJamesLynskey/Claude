Claude Cowork

Autonomous Agents in Cloud VMs

Hand Claude a task and walk away. Each session is a real VM with browser, terminal, Python, Node, and Git — driven autonomously.

Cowork Cloud VMs Dispatch
01

What It Is

Cowork is the autonomous tab of Claude Desktop. Hand it a goal; it runs in an Anthropic-hosted VM until done.

Is

  • Agent loop in a real VM
  • Detached — keeps running if you close the app
  • Live video feed of progress
  • Persistent workspace for the task

Is Not

  • Not a server you SSH into
  • Not running on your machine
  • Not the Code tab
  • Not offline

Core idea: give Claude a body — a computer it can use — so you can delegate tasks that don’t fit in a chat turn.

Starting a Task

1. Goal

  • Describe the outcome, not the steps
  • State success criteria
  • Paste URLs / ticket numbers

2. Context

  • Drag files → land in ~/workspace
  • Link a Project
  • Enable connectors

3. Model & Mode

  • Opus / Sonnet / Haiku
  • Extended thinking
  • Ask-before-risky toggle

4. Launch

  • VM spins up in seconds
  • Live feed on the right
  • Close the app — it keeps going
04

Inside the VM

Computer Use

  • Mouse, keyboard, vision
  • Real browser — fills forms
  • Screenshots at any step

Shell & Dev

  • Terminal, git, builds
  • Install pkgs (apt, pip, npm)
  • VS Code server

Data

  • Python, pandas, matplotlib
  • Read/write ~/workspace

Can’t

  • Reach your localhost, LAN, or private network
  • Run stdio MCP servers from your machine
  • Connect to plain HTTP or self-signed cert endpoints
  • Share state between tasks · GPU workloads

06

Dispatch

Mobile entry point. Queue Cowork tasks from the phone.

How

  • Claude mobile app → Dispatch tab
  • Type or dictate a goal
  • Push notification on completion
  • Review transcript on desktop later

Why

  • Capture tasks while away from the machine
  • Voice-first while walking / commuting
  • Chain work during meetings

Templates

  • Save prompts as one-tap templates
  • Sync with desktop

Limits

  • No live video on mobile
  • Can’t take VM control from phone
  • Long approvals better on desktop
07

Parallel Coworkers

Each task is its own VM. Run many at once — colour-coded cards in the sidebar.

Status

  • Green running · Amber waiting
  • Blue paused · Purple done
  • Red failed / stopped

Patterns

  • Fan-out research (many angles, one report)
  • Watchers — ping on change
  • Pipelines — A produces, B consumes
  • PR bots per repo feeding Slack

Heads-up: coworkers don’t share state. Wire them via a connector (Drive, Git) or copy the artefact yourself.

09

Security & Limits

Posture

  • Ephemeral VMs
  • No tokens on VM disk
  • Per-tool allow-lists
  • Outbound via Anthropic network

Risks

  • Prompt injection from browsed pages
  • Never paste passwords — use connectors
  • Auto-approve can loop — set limits

Cost

  • Included in Pro/Max
  • Per-session + per-day caps
  • Live token / tool count

Hard Limits

  • Max wall-clock per task
  • Max concurrent tasks
  • No GPU, no inbound, no LAN
  • No custom VM image
10

Good / Bad Tasks & Tips

Great Fit

  • Multi-source research writeups
  • Clone, test, open PR
  • Web scraping public data
  • Scheduled watchers
  • Data cleanup in pandas

Bad Fit

  • Needs your local files
  • GPU / heavy compute
  • Interactive pair programming
  • Strict bot-detect sites

Prompting

  • State deliverable and where it goes
  • Success criteria
  • Preferred tools
  • Time budget
  • Failure behaviour

Stuck Task

  • Scrub feed backwards
  • Take control, do the step, hand back
  • Fork at last good checkpoint
