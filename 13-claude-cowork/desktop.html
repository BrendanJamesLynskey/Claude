Claude Cowork

Autonomous Background Agents in Anthropic-Hosted Cloud VMs

Hand Claude a task and walk away. Each Cowork session is a real virtual machine — browser, terminal, Python, Node, Git — that Claude drives autonomously while you watch live, or ignore, or queue up five more. Pair it with Dispatch and you can kick off the same task from your phone.

01

What Cowork Is — and What It Isn’t

Cowork is the autonomous mode of the Claude Desktop app. Unlike Chat (stateless turn-by-turn) or Code (you approve every edit), a Cowork session is handed a goal and then runs on its own inside a cloud-hosted virtual machine until it finishes, asks for input, or hits its budget.

What It Is

  • An agent loop running inside a real VM — filesystem, network, desktop environment
  • Detached from your machine — close the app, it keeps running
  • Live video feed so you can watch it click, type, and browse
  • Persistent workspace for the task’s lifetime
  • Composable — tasks can trigger other tasks, or report back on a schedule

What It Is Not

  • Not a rented server you SSH into — Claude is the only user
  • Not a container on your machine — it runs in Anthropic’s cloud
  • Not the Code tab — Cowork is for goals you’d rather not supervise
  • Not the API’s managed-agent endpoint — this is the consumer-product surface
  • Not offline — a network and your Anthropic account are required

Core idea: Cowork gives Claude a body — a computer it can actually use — so you can delegate tasks that don’t fit in a chat turn. Research, repo work, data wrangling, triage, PR monitoring, content drafting from web sources. The desktop app is just the viewport onto that remote body.

Plans

  • Free — no Cowork
  • Pro — Cowork with per-session limits
  • Max — higher concurrency, longer sessions
  • Team / Enterprise — pooled quota, admin controls, audit log

Network

Your machine only needs outbound HTTPS to Anthropic. The VM’s own outbound network goes through Anthropic’s infrastructure — so any site that allow-lists by IP will see an Anthropic range, not yours.

The VM Image

  • Linux desktop environment (browser, terminal, file manager)
  • Pre-installed: git, python, node, npm, curl, jq, ffmpeg, common build tools
  • Headless Chromium for browsing
  • Ephemeral root filesystem with a persistent ~/workspace
  • Snapshots taken when the task pauses for your input

Budgets & Timeouts

  • Each task has a plan-dependent compute budget (wall-clock + tool calls)
  • Tasks auto-pause when the budget is nearly spent and ask if you want to continue
  • Idle sessions are hibernated; resuming restores the VM state

Rollout Note

Cowork is still rolling out region-by-region. If the tab is missing on a supported platform and plan, check Settings → Account for region status.

03

Starting a Cowork Task

The Cowork tab has a prompt box, a model picker, and an attach button. That’s it. Everything else happens after you hit send.

1. Write the Goal

  • Describe the outcome, not the steps (“produce X” beats “open Y, click Z”)
  • State success criteria — how will you know it’s done?
  • Paste URLs, file excerpts, or ticket numbers as context
  • Mention credentials or connectors the task will need

2. Attach Context (Optional)

  • Drag files from your machine — they land in ~/workspace
  • Link a Project — its files and system prompt apply
  • Enable connectors: GitHub, Linear, Slack, Gmail, Google Drive

3. Model & Mode

  • Model picker — Opus for hard reasoning, Sonnet for most work, Haiku for fast jobs
  • Extended thinking toggle for multi-step research
  • “Ask before risky actions” vs “run until done”

4. Launch

  • Click send — a VM spins up in a few seconds
  • Live desktop feed appears on the right
  • Chat transcript on the left mirrors Claude’s reasoning and tool calls
  • You can close the window — the task keeps running
Good: “Clone github.com/acme/pipeline, run the test suite,
       open a PR that fixes any failing tests, and post a
       summary in #eng-notes when done.”

Less good: “Fix the tests.”
04

Inside the VM — What Claude Can Do

Computer Use

  • Move the mouse, click, type, scroll
  • Read the screen with vision
  • Drive a real browser — fill forms, click through modals, handle cookies
  • Screenshot the desktop at any step

Shell & Dev Tools

  • Open a terminal — bash, zsh
  • Clone repos, run builds, start servers
  • Install packages (apt, pip, npm)
  • Edit files with vim or VS Code server

Data & Files

  • Python / Jupyter for analysis
  • pandas, numpy, matplotlib pre-installed
  • Read and write to ~/workspace
  • Export results back to your machine via the chat transcript

Connectors Projected Into the VM

Any connector you’ve authorised in the desktop app is visible inside the VM as a tool call. Cowork doesn’t install a browser extension or store your tokens on the VM disk — each call is brokered through Anthropic’s connector layer, so revoking a connector in Settings immediately cuts off in-progress tasks from it.

What It Can’t Do

  • Reach your local network / localhost unless you tunnel it in
  • Persist state between unrelated tasks (each task gets a fresh VM)
  • Run GPU workloads — the VM is CPU-only
  • Bypass the connector permission layer — every external write is still gated
05

Monitoring, Pausing, and Resuming

The Live Feed

  • Right pane shows the VM desktop in real time
  • Left pane shows Claude’s reasoning and tool calls
  • Click on a tool call to jump to the moment in the feed
  • Scrub backwards to see what Claude saw before a decision

Interrupting

  • Stop — kills the task and snapshots the VM
  • Pause — halts after the current tool call
  • Message — type a correction; Claude resumes with it in context
  • Take control — grab the mouse/keyboard yourself, then hand back

When Claude Pauses Itself

  • Risky action pending — delete, send, publish, pay
  • Credential or 2FA challenge in a web flow
  • Ambiguous instruction — asks for clarification
  • Budget running low — asks before continuing
  • Desktop app surfaces an OS notification so you don’t miss the prompt

Resuming Later

  • Close the app and come back — task history is on the left sidebar
  • Click a paused task — VM rehydrates in seconds from its snapshot
  • Past tasks can be forked into a new session with the same workspace state

Dispatch — Kick Off Cowork From Your Phone

Dispatch is the mobile entry point to Cowork. It lives inside the claude.ai mobile apps and lets you queue up autonomous tasks without having to open the desktop.

How It Works

  • Open the Claude mobile app → Dispatch tab
  • Type a goal or dictate with voice
  • Tap send — Anthropic provisions a Cowork VM
  • Push notification when the task is done or needs input
  • Open the desktop app later to see the full transcript and artefacts

Why It Matters

  • You’re away from your machine and remember something — “research X by tonight”
  • You want to chain work: Cowork runs a 30-minute task while you’re in a meeting
  • Voice-first task capture while walking, commuting, or driving

Templates

  • Save a prompt as a template — “daily news digest”, “PR triage”, “weekly status”
  • Tap once to launch without retyping
  • Templates sync with desktop — pick them up anywhere

Limits vs Desktop

  • No live video feed on mobile — status updates only
  • You can’t take control of the VM from the phone
  • File drops are awkward — use iCloud / Drive / connectors instead
  • Long approval flows are best handled on desktop
07

Parallel Coworkers

Because each task is its own VM, you can have many of them running at once — each in its own isolated environment, each pursuing a different goal. The desktop app surfaces them as cards on the left sidebar, colour-coded by status.

Status Colours

  • Green — running
  • Amber — waiting for you
  • Blue — paused (budget)
  • Purple — completed
  • Red — failed or stopped

Concurrency Limits

  • Pro: small handful of concurrent tasks
  • Max: more concurrent tasks and longer runs
  • Team / Enterprise: pooled across the org
  • Hard cap per account to prevent runaway spend

Patterns

  • Fan-out research: five coworkers, five angles, one report
  • Watcher tasks: one coworker polls and only pages you on change
  • Pipelines: task A produces a file, task B consumes it
  • PR bots: a coworker per repo, all feeding one Slack channel

Heads-up: coworkers do not share state by default. If you want task B to see task A’s output, either wire them up through a connector (drop to Drive, write to a GitHub branch) or copy the artefact into task B’s prompt yourself. A shared workspace is on the roadmap but isn’t available today.

09

Security, Cost, and Limits

Security Posture

  • VM is ephemeral — destroyed when the task is archived
  • No connector tokens written to the VM disk
  • Connectors enforce per-tool allow-lists — risky writes still prompt you
  • Audit log of every tool call (Enterprise)
  • VM outbound traffic goes via Anthropic infrastructure

Risks to Manage

  • Prompt-injection from web content the VM browses
  • Credential prompts in a web flow — never paste passwords into Cowork prompts; use connectors
  • Auto-approve of tool calls can produce unintended writes — keep “ask before risky” on for important tasks
  • Long-running tasks can burn budget on loops — set a wall-clock limit

Cost & Budgets

  • Pro / Max: included in the plan, subject to per-session and per-day caps
  • Team / Enterprise: pooled quota, org-admin can set per-user caps
  • Cowork sessions show live token and tool-call counts in the sidebar
  • Hitting the cap pauses the task and asks you to approve continuation

Hard Limits (at time of writing)

  • Max wall-clock per task (plan-dependent — minutes to hours)
  • Max concurrent tasks per account
  • No GPU, no inbound network, no access to your LAN
  • No custom base VM image — the OS image is managed by Anthropic

Good Tasks, Bad Tasks, and Prompting Tips

Great Fit

  • Multi-source research with a written deliverable
  • Repo cloning, test running, PR opening
  • Form filling, account setup, web scraping of public data
  • Scheduled “watcher” tasks that only ping you on change
  • Data cleanup in Python / pandas on files you upload
  • Drafting content from browsed sources

Bad Fit

  • Anything needing access to your local filesystem — use the Code tab
  • Heavy compute (GPU, big training runs)
  • Interactive pair-programming — the feedback loop is too slow
  • Sites with strict bot detection — Cowork looks like a headless browser
  • Anything you’d struggle to describe in a prompt

Prompting Tips

  • State the deliverable and where it should end up (file name, channel, PR)
  • Give success criteria: “done when the failing tests pass and CI is green”
  • List the tools Claude should prefer: “use the GitHub connector, not the CLI”
  • Budget guidance: “give up after 20 minutes and summarise what you learned”
  • Attach any data once rather than pasting it over and over
  • Set failure behaviour: “if you get stuck, ask before retrying”

Debugging a Stuck Task

  • Scrub the live feed backwards to the last visible progress
  • Take control of the VM, do the next step manually, hand back
  • Pause and paste a clarifying message
  • Fork the task at the last good checkpoint instead of restarting from zero
