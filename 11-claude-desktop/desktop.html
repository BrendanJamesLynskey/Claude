01

What Is Claude Desktop?

Claude Desktop is the native application for talking to Claude on your own computer. It is a thin, signed-in wrapper around the same backend that powers claude.ai, but with a handful of things the browser cannot give you — a persistent tray/menu-bar icon, a global keyboard shortcut, native file-picker and drag & drop, desktop notifications, and (via connectors) access to local and remote tools.

Chat

  • General conversation
  • Projects, Artifacts, Styles
  • Files, images, PDFs
  • Extended thinking
  • Voice mode (where available)

Cowork

  • Autonomous background agent
  • Runs in an Anthropic-hosted VM
  • Works while you do something else
  • Dispatched from desktop or phone

Code

  • Interactive pair-programming assistant
  • Direct access to local files
  • Visual diffs, parallel Git worktrees
  • Same engine as the Claude Code CLI

Why a native app? You get system-wide integration: a global hotkey to summon Claude from anywhere, a menu-bar / tray icon for quick access, native file dialogs, OS-level notifications when a long-running Cowork task finishes, and installable connectors that plug into local tools (filesystem, iMessage, calendars) or remote services (GitHub, Slack, Gmail) via MCP.

Signing In

  1. Launch the app — browser opens for OAuth
  2. Sign in with the same account as claude.ai
  3. Approve the redirect back to the desktop app
  4. Your plan, Projects, and chat history all follow the account

Plans & Tabs

  • Free — Chat only, limited model and rate
  • Pro / Max — Chat + Cowork + Code; higher limits; extended thinking
  • Team / Enterprise — everything plus org admin, SSO, audit

Auto-Update

The app checks on launch and installs updates in the background. A blank screen or a stale tab after a release is almost always fixed by a full restart.

03

The Chat Tab — Conversation, Projects, Artifacts

Chat is the default tab and the one most people will use every day. It mirrors claude.ai feature-for-feature but adds the native ergonomics of a real app.

Conversations & Projects

  • Left sidebar lists recent chats, starred chats, and Projects
  • A Project bundles a system prompt, uploaded files, and its own chats — great for a book, a codebase, a client
  • Synced with the web — start on desktop, continue on phone

Artifacts

  • Claude renders long outputs (code, HTML, SVG, Markdown, React components) into a side pane
  • Live preview for HTML and React — you can click around inside
  • Versioned — earlier drafts are one click away

Input Modalities

  • Drag files from Finder / Explorer — .pdf, .docx, images, code, CSVs
  • Screenshot paste (⌘V / Ctrl+V straight from the clipboard)
  • Voice dictation (OS-level on macOS and Windows)
  • Multi-file uploads up to the plan’s context limit

Model & Thinking Controls

  • Model picker next to the send button: Opus, Sonnet, Haiku
  • Extended thinking toggle for harder problems
  • Styles — concise, explanatory, formal, or your own writing sample
  • Per-message regenerate, edit, and branch

Desktop-only niceties: a global hotkey (configurable) to summon a new chat from any app, a menu-bar / tray popover for a quick question without switching windows, and native OS notifications when a long response finishes while you’re in another app.

04

The Cowork Tab — Autonomous Background Agents

Cowork is where you hand Claude a task and walk away. Each task runs inside an Anthropic-hosted cloud VM with its own filesystem, network, and tools — independent of your machine. When the task finishes (or needs your input), the desktop app surfaces a notification.

Typical Uses

  • “Research the top five open-source vector DBs and write a comparison”
  • “Clone this repo, set up the project, run the tests, and open a PR for the failing ones”
  • “Monitor this PR queue and comment on anything that breaks CI”
  • “Draft a weekly status email from these Linear tickets”

What You Get for Free

  • Browser, terminal, Python, Node, Git — pre-installed
  • Persistent workspace for the task’s lifetime
  • Live video of the VM so you can watch progress
  • Dispatch integration — kick off the same task from your phone

Platform Caveat

On macOS the Cowork tab currently requires Apple Silicon (M1 or later). On Intel Macs the Chat and Code tabs still work, but Cowork is hidden. On Windows, Cowork is available on all supported hardware.

→ Presentation 13 is the dedicated deep dive on Cowork — architecture, session lifecycle, Dispatch, parallel agents, costs, and limits.

05

The Code Tab — Interactive Pair Programming

Code is the graphical interface to the same engine that runs the claude CLI. You pick a local folder (or a remote machine over SSH, or an Anthropic cloud session), and Claude edits files, runs commands, and reviews diffs with you.

What Makes It Graphical

  • Side-by-side visual diffs with inline comments
  • Permission prompts as dialogs, not terminal questions
  • Live app preview pane for web projects
  • Parallel sessions automatically isolated in Git worktrees

Shared With the CLI

  • CLAUDE.md project instructions
  • Hooks, sub-agents, memory, skills
  • Slash commands and keybindings
  • MCP servers configured once and reused

Environments

  • Local — your machine, your files
  • Remote — Anthropic cloud, continues if you close the app
  • SSH — a server you control, Claude Code installed remotely

→ Presentation 12 covers the Code tab in depth — permission modes, diff review workflow, parallel sessions, PR monitoring, and computer use.

06

Connectors, Skills, and Tools

The desktop app exposes Claude’s tool layer directly in the UI. You add capabilities without touching config files.

Connectors (MCP)

  • Installable via Settings → Connectors
  • Local: filesystem, iMessage, calendar, shell
  • Remote: GitHub, Linear, Slack, Gmail, Google Drive, Notion
  • Each connector is an MCP server under the hood — the same protocol the CLI uses
  • Per-connector permission prompts the first time a tool is called

Skills

  • Bundled procedures the app can invoke on your behalf
  • Examples: PDF generation, Excel, slide decks, diagram rendering
  • Managed from Settings → Skills — toggle on or off

Built-in Tools

  • Web search — on by default for recent-info questions
  • File analysis — Claude reads uploaded docs, images, and data
  • Code execution in a sandbox for Chat and Cowork
  • Computer use (Cowork & Code) — Claude drives a virtual desktop

Permission Model

  • First call of a new tool → dialog asks you to allow
  • Allow once, allow this session, or allow always
  • Revoke any time from Settings → Permissions
  • Org admins can lock the allow-list in enterprise plans

Privacy, Data, and Enterprise Controls

Data Defaults

  • Consumer plans (Free / Pro / Max): conversations are not used for model training by default
  • Team / Enterprise: contractually excluded from training
  • Chat history synced to Anthropic and retained per your plan’s policy
  • Cowork VMs are ephemeral — destroyed when the task is archived

What the App Sends

  • Your prompts and any files you attach
  • Tool-call inputs and outputs (for diagnostics and billing)
  • Crash reports and anonymised telemetry — opt-out in Settings
  • OAuth token for your account — stored in OS keychain

Enterprise Configuration

  • SSO via SAML or OIDC — Okta, Google, Entra
  • SCIM user provisioning
  • Admin console: model allow-list, connector allow-list, data-retention window
  • Audit log of every chat, tool call, and file access
  • DLP integrations via the API for outbound content inspection

Managed Deployment

  • Silent install via MDM (Jamf, Intune)
  • Config profile can lock the global shortcut, disable Cowork, or pin a region
  • AWS Bedrock and GCP Vertex routes available for regulated customers

Troubleshooting & Common Gotchas

Sign-in Loops

  • Kill the app fully, re-open, click “Sign in” again
  • Clear the OAuth token from the OS keychain if the browser keeps redirecting
  • 403 or “unauthorised” — sign out from the app menu and back in

Cowork Tab Missing

  • macOS: check you’re on Apple Silicon (uname -m → arm64)
  • Check your plan — Free plans don’t see Cowork
  • Region: Cowork availability is rolling out by region — check Settings → Account

Connectors Won’t Install

  • Local connectors need the target app installed (e.g. iMessage)
  • Remote connectors need you to OAuth into the service in a browser
  • Corporate networks may block the MCP WebSocket — check with IT

Performance & Hygiene

  • Blank screen after update → quit fully and relaunch
  • High memory → too many open chats; close unused tabs
  • Logs: ~/Library/Logs/Claude/ (mac) / %APPDATA%\Claude\logs (win)
  • Report bugs from Help → Send Feedback — attaches logs automatically

MCP Across the Claude Product Range

The Model Context Protocol is the common tool-server protocol used by every Claude surface, but how much of it you get depends on where the model runs and whether Claude can spawn local processes on your machine. Two flavours matter: local stdio MCP servers (a subprocess on your box talking over stdin/stdout) and remote HTTP MCP servers (Anthropic-brokered Connectors, or SSE/HTTP endpoints on the network).

Surface Local stdio MCP Remote HTTP MCP Config / how to add Notes
Claude Code CLI Yes Yes claude mcp add · ~/.claude.json · project .mcp.json Full host. Per-user, per-project, and local scopes. Hooks and skills apply.
Desktop — Chat tab Yes Yes (as Connectors) claude_desktop_config.json · Settings → Connectors Original MCP launch surface (Nov 2024). Config file is separate from the Code tab.
Desktop — Code tab Yes Yes Reads the same files as the CLI (~/.claude.json, .mcp.json) Chat-tab MCP servers do not appear here, and vice versa.
Desktop — Cowork tab No Yes (brokered) Connectors you’ve authorised in the app Code runs in an Anthropic-hosted cloud VM, so it can’t reach stdio on your machine.
claude.ai (web) No Yes (Connectors) Settings → Connectors in the web UI Browser sandbox — no local processes. Custom remote MCP on Team / Enterprise.
Claude mobile apps No Yes (Connectors) Managed account-wide Connector catalogue is a subset of what desktop sees.
Messages API No (not directly) Yes mcp_servers parameter; or run your own MCP client Bring-your-own: the API forwards to HTTP MCP servers you point it at.

Rule of Thumb

If a tool is a local process (filesystem, iMessage, a database on localhost, a shell script), only the three runs-on-your-machine surfaces can use it: Claude Code CLI, the Chat tab, and the Code tab. Everything cloud — Cowork, web, mobile, the API — needs an HTTP endpoint or an Anthropic-hosted Connector.

Common Pitfalls

  • Configuring an MCP server in claude_desktop_config.json and wondering why the Code tab can’t see it — they’re separate files.
  • Expecting Cowork to run a stdio MCP binary from your laptop — it can’t; publish it as a remote MCP server or wrap it in a Connector.
  • Corporate networks blocking outbound WebSocket → Connectors silently fail to install. Check firewalls.
  • Permission dialogs for MCP tool calls stack with the app’s own permissions — both have to allow.
