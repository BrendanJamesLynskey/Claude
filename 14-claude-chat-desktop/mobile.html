Claude Chat Desktop

The Chat Tab in the Claude Desktop App

Projects, Artifacts, Styles, Connectors, and the original MCP surface via claude_desktop_config.json.

Chat Projects Connectors Local MCP
01

What It Is

The Chat tab is claude.ai in a native window: same account, same models, same history. The native shell adds ergonomics and local reach.

Is

  • Turn-by-turn conversation, synced
  • Projects, Artifacts, Styles
  • Extended thinking, voice
  • Local stdio MCP via claude_desktop_config.json

Is Not

  • Not a file editor (Code tab does that)
  • Not autonomous (Cowork does that)
  • Not a separate product from claude.ai
  • Not plan-locked — Free plans get it

Mental model: Claude for you, not Claude for your repo.

03

Projects

A Project = system prompt + file library + chat history. Every chat in the Project sees all three.

Contents

  • Custom instructions (system prompt)
  • Knowledge: PDFs, docs, images, code
  • Pinned chats

Common Shapes

  • Book / manuscript
  • Client dossier
  • Read-only codebase reference
  • Research dossier

Sharing

  • Personal = private
  • Team / Enterprise: share with org
  • Shared Projects: shared Knowledge, private chats

Limits

  • Plan-dependent file count / size
  • Very large Knowledge is retrieved, not attached in full
  • Re-upload replaces files (no diff)
04

Artifacts

Substantial outputs lift out of the chat into a dedicated side pane with live preview.

Types

  • HTML / CSS / JS (live preview)
  • React components (sandboxed)
  • SVG, Mermaid, Markdown
  • Code in any language

What You Can Do

  • Interact with the preview
  • Copy / download
  • Ask to update — new version
  • Jump between versions

Publishing

  • Public link (Pro / Max)
  • Recipients see rendered preview
  • Frozen or live

Desktop advantage: preview pane accepts drag-and-drop and clipboard. For actual file writes, use deck 12.

07

Connectors & MCP

The Chat tab is the original MCP surface (Nov 2024).

Connectors (GUI)

  • Settings → Connectors
  • Local: filesystem, iMessage, calendar
  • Remote: GitHub, Linear, Slack, Gmail, Drive
  • Each is MCP under the hood

Skills

  • PDF, Excel, slides, diagrams
  • Auto-selected or toggled

Raw MCP config

  • macOS: ~/Library/Application Support/Claude/claude_desktop_config.json
  • Win: %APPDATA%\Claude\claude_desktop_config.json
  • Supports stdio (command + args) and Streamable HTTP (url)
  • Restart the app after editing

HTTP & Private Networks

  • Chat tab connects from your machine — plain HTTP and private IPs work
  • E.g. http://10.60.12.34:3001/mcp
  • HTTPS only needed for Anthropic-brokered surfaces (Cowork, Web, Mobile, API)
  • Chat-tab config is separate from Code-tab — servers don’t cross. See deck 11.

10

Good Uses & Tips

Great Fit

  • Writing and editing in a Project
  • Explaining code, papers, docs
  • Data analysis on pasted tables
  • “What is this?” screenshots
  • Research via Connectors
  • Voice brainstorming
  • Power-user local stdio MCP

Bad Fit

  • Editing repo files (→ deck 12)
  • Autonomous work (→ deck 13)
  • Scripting (→ API / CLI)

Prompting Tips

  • Durable context → Project instructions
  • Authoritative docs → Knowledge
  • Ask for artefacts explicitly
  • Styles per venue
  • Branch, don’t re-correct

Housekeeping

  • Archive finished chats
  • Prune Knowledge — tokens cost every turn
  • Rename chats for search
  • Keep Chat / Code MCP configs separate
