Claude Chat Desktop

The Chat Tab in the Claude Desktop App

The everyday Claude — same backend as claude.ai, same Projects, same Artifacts, but with a global hotkey, menu-bar popover, drag-and-drop, native notifications, and the original stdio MCP surface via claude_desktop_config.json. The oldest of the three Desktop tabs; the first place MCP ever ran.

01

What the Chat Tab Is

The Chat tab is the part of Claude Desktop that most people open first and use most often. Functionally it is claude.ai in a native window: the same account, the same models, the same chat history, the same Projects and Artifacts. What the native shell adds is ergonomics and local reach — things a browser tab structurally can’t do.

What It Is

  • A turn-by-turn conversation with Claude, synced to your account
  • Projects: persistent system prompts, uploaded files, per-project chat history
  • Artifacts: code, HTML, SVG, Markdown, React rendered in a side pane
  • Extended thinking, Styles, Voice, model picker
  • The original MCP launch surface — Nov 2024 — via claude_desktop_config.json

What It Is Not

  • Not a file editor — it cannot write to disk on its own (that’s the Code tab)
  • Not an autonomous agent — every turn is your turn (that’s Cowork)
  • Not a separate product from claude.ai — same backend, same history
  • Not restricted to any plan — Free plans get the Chat tab too

Mental model: the Chat tab is Claude for you, not Claude for your repo. It draws on whatever Claude knows, whatever you put in a Project, whatever a Connector can fetch, and whatever you paste or drop into the prompt. It doesn’t touch your filesystem — but via local stdio MCP servers it can reach almost anything else on your machine.

03

Projects — Persistent Context, Files, and Instructions

A Project is a named workspace that combines a system prompt, a file library, and a chat history. Every chat started inside a Project sees all three. It is the single biggest reason the Chat tab feels different from a raw model playground.

What You Put Into a Project

  • Custom instructions — a system prompt that applies to every chat (tone, constraints, goals)
  • Knowledge — uploaded PDFs, docs, spreadsheets, images, code, audio transcripts
  • Pinned chats — the ongoing conversations you want at the top of the sidebar

Common Shapes

  • “Book I’m writing” — manuscript chapters, style guide, character bible
  • “Client X” — contract, meeting notes, product brief, prior emails
  • “My codebase (read-only)” — design docs, architecture notes; no write access (use the Code tab for that)
  • “Research dossier” — papers, benchmarks, previous summaries

How It’s Used Each Turn

The system prompt is prepended as the project’s custom instructions. Knowledge files are retrieved by relevance or fully attached (depending on size and plan). Your chat history provides the usual turn-by-turn context. The effective prompt can be very large — watch the token meter on Pro / Max.

Sharing & Permissions

  • Personal projects are private to your account
  • Team / Enterprise plans can share projects with an org or a group
  • Shared projects have separate per-user chat history — Knowledge is shared, conversations are not
  • Admin console controls which users can create or view projects

Limits

  • Per-project file count and total size depend on plan
  • Very large Knowledge sets get retrieved rather than fully attached — keep critical text in the custom instructions
  • Files are versioned but not diffed — re-uploading replaces the previous copy
04

Artifacts — Live Preview in a Side Pane

When Claude produces a substantial piece of output — a block of code, an HTML page, a Mermaid diagram, a Markdown document, a React component — the desktop app lifts it out of the chat into a dedicated Artifacts pane. The conversation stays on the left; the artefact and its preview are on the right.

Renderable Types

  • HTML / CSS / JS (with live preview)
  • React components (preview sandboxed)
  • SVG (rendered inline)
  • Mermaid diagrams (rendered inline)
  • Markdown (rendered inline)
  • Plain code in any language (syntax highlighted)

What You Can Do

  • Interact with the preview — click, type, watch state change
  • Copy the artefact to your clipboard
  • Download it as a file
  • Ask Claude to “update the artifact” — it produces a new version
  • Jump back to earlier versions via the version switcher

Publishing

  • Share an artefact via a public link (Pro / Max only)
  • Recipients see the rendered preview — no account required
  • Published artefacts update when you share again, or stay frozen if you prefer

Why it matters for the desktop app specifically: the preview pane is an embedded browser, and on desktop it can load drag-and-dropped assets, paste from the system clipboard, and use OS-level shortcuts. Iterating on a React prototype feels much closer to a real dev loop than on the web — though Claude still doesn’t touch your filesystem from this tab. For that you want deck 12.

05

Input — Text, Files, Images, Voice

Text

  • The prompt box accepts Markdown — code blocks, lists, tables render on send
  • Shift+Enter for a newline; Enter to send
  • Paste any length of text — the app auto-wraps very long pastes into an inline attachment so the prompt stays readable
  • Edit a previous message to branch the conversation

Files

  • Drag from Finder / Explorer straight onto the window
  • Supported: .pdf, .docx, .pptx, .xlsx, .csv, .md, .txt, most source code, .json, .yaml
  • Images: PNG, JPEG, WebP, GIF, SVG — Claude sees them via vision
  • Audio transcripts can be dropped in as text
  • Per-plan file count and total size caps per message

Screenshots & Clipboard

  • ⌘Shift+4 / PrtSc → paste straight into the prompt
  • Rich text from the clipboard is preserved (tables, code blocks, links)
  • Images from Preview, Figma, browser tabs — all accepted

Voice

  • OS-level dictation works in the prompt field
  • Voice Mode (where available) — push-to-talk, Claude replies with speech
  • Handy for hands-free use while walking through a problem aloud

What You Can’t Drop In

  • Folders — only individual files (use a Project for bulk uploads)
  • Files larger than the plan’s per-upload cap
  • Executables (they’re ignored for safety)
  • Live links to cloud files — upload a snapshot or use a Connector
06

Models, Extended Thinking, and Styles

Model Picker

  • Next to the send button
  • Switch between Opus, Sonnet, Haiku
  • Choice is per-message, not per-chat — you can mix
  • Free plan is limited to the smaller models

Extended Thinking

  • Toggle in the prompt bar on Pro and above
  • Claude produces a visible scratchpad before the final answer
  • Latency is higher; quality on hard problems is noticeably better
  • On Opus / Sonnet 4.6 thinking depth is adaptive — you don’t tune a budget

Styles

  • Preset styles: Formal, Concise, Explanatory, Normal
  • Custom styles: paste a writing sample, Claude matches its voice and tone
  • Per-chat; can be changed mid-conversation
  • Styles are global (apply inside Projects too)

Tips for Choosing

  • Haiku — short questions, quick classification, bulk rewriting — cheapest and fastest
  • Sonnet — default for most chats: writing, analysis, coding help, research summaries
  • Opus + extended thinking — hard multi-step reasoning, long planning, rigorous technical work
  • Don’t forget to downshift: a follow-up “now make it shorter” doesn’t need Opus
07

Connectors, Skills, and the Chat-Tab MCP Surface

The Chat tab is the original Claude MCP surface. Long before the Code tab existed, the Claude Desktop app let you point Claude at local stdio MCP servers via a config file — that’s how MCP launched in November 2024. That surface is still here, and it is the reason the Chat tab punches above its weight for power users.

Connectors — the GUI Path

  • Settings → Connectors — a curated gallery of integrations
  • Local: filesystem, iMessage, calendar, shell wrappers
  • Remote: GitHub, Linear, Slack, Gmail, Google Drive, Notion
  • Each Connector is an MCP server under the hood — Anthropic just hides the wiring
  • Per-tool permission prompts on first use; revoke any time

Skills

  • Bundled procedures — PDF generation, Excel, slide decks, diagrams
  • Toggle from Settings → Skills
  • Claude picks skills automatically when they fit the task
  • Skills and Connectors stack — a Skill can call a Connector

Raw MCP via claude_desktop_config.json

For servers not in the Connectors gallery — your own stdio server, a custom tool, a third-party integration — edit the config file directly:

  • macOS: ~/Library/Application Support/Claude/claude_desktop_config.json
  • Windows: %APPDATA%\Claude\claude_desktop_config.json

Supports both transport types:

  • stdio — "command" + "args" — Claude spawns the process locally
  • Streamable HTTP — "url" pointing to an HTTPS endpoint with a CA-signed certificate
// stdio example (local process)
{
  "mcpServers": {
    "filesystem": {
      "command": "npx",
      "args": ["-y", "@modelcontextprotocol/server-filesystem",
               "/Users/me/Documents"]
    }
  }
}

// Streamable HTTP — HTTP or HTTPS; private IPs work
{
  "mcpServers": {
    "lan-api": {
      "url": "http://10.60.12.34:3001/mcp"
    },
    "public-api": {
      "url": "https://mcp.example.com/v1/mcp"
    }
  }
}

Separate Config Files — The Biggest Gotcha

MCP servers listed in claude_desktop_config.json are visible to the Chat tab only. The Code tab reads ~/.claude.json and per-project .mcp.json (shared with the CLI). If a server appears in one tab but not the other, that’s why. See deck 11 for the full MCP matrix.

HTTP & Private Networks

  • The Chat tab runs on your machine and connects directly — plain HTTP and private IPs work
  • http://192.168.1.50:8080/mcp, http://localhost:3000/mcp, http://10.x.x.x:3001/mcp — all valid
  • HTTPS is only required for Anthropic-brokered surfaces (Connectors UI, Cowork, Web, Mobile, API)
  • Restart the app after editing the config file

The boundary with Code: if you’re asking Claude to change files in a repo you have open, use the Code tab. Good Uses, Limits, and Prompting Tips

Great Fit

  • Writing, editing, and polish on drafts in a Project
  • Explaining code, concepts, papers, dense documents
  • Data analysis on pasted or uploaded tables (Python in artefacts)
  • Quick screenshots to ask “what is this?” or “fix this error”
  • Research assistance with Connectors (search, Drive, Notion)
  • Voice-first brainstorming while walking
  • Power-user local tools via stdio MCP — SQLite, Obsidian, filesystem, custom scripts

Bad Fit

  • Editing files in your repo — use the Code tab (deck 12)
  • Multi-step autonomous work — use Cowork (deck 13)
  • Anything that needs to write to arbitrary paths on disk
  • Scripting / automation — use the API or the CLI

Prompting Tips

  • Put durable context (style, constraints, references) in the Project’s custom instructions, not in every prompt
  • Upload the authoritative version of a doc to Knowledge — don’t paste snippets turn after turn
  • Ask for artefacts explicitly when you want a rendered result: “produce an HTML artifact that…”
  • Use Styles to match a venue — formal for board reports, concise for Slack
  • When a chat drifts, branch off an earlier message instead of correcting turn after turn

Housekeeping

  • Archive finished chats to keep the sidebar readable
  • Periodically prune Knowledge files that are no longer relevant — they cost tokens every turn
  • Rename chats so search (⌘K) actually finds them
  • Keep the Chat-tab and Code-tab MCP configs consciously separate — don’t try to share them
